## 盒子模型
- 文档流 
  doctype 
  从上到下(块)，从左到右，像水流一样显示我们的页面  流体布局
  html 标签 从外到内（层级关系），从上到下（布局 盒子们） 从左到右（布局）
- html 元素的时候 ，一个盒子
- 盒子 页面的占位就清楚了。
  - 盒子的尺寸 由 哪些部分
  - 内容 width x height
  - 内边距 padding
  - 边距 border
  - 外边距 margin
  - 
- 页面是什么显示的？
    - 将元素和css，了解他作为一个盒子，在文档流中的大小和位置
    - 盒子模型 = 内容 + 内边距 + 边框 + 外边距
    两种算法 
- z-index 
    子元素的z-index会被父元素的z-index给覆盖
- z-index不能乱设置
